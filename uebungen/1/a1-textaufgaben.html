Fragen zur ersten Uebung

1.1. Fachliche Argumentation über Erfolgsprinzipien des WWW

  1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?

    1. Die Hypertext Community stört sich an den möglichen broken links vom WWW. Dies sah man als Rückschritt von referenzielle Integrität
    2. Die Evolvierbarkeit von HTML wurde als Wagheit missverstanden
    3. Wunsch nach verteilten Objekten, statt verteilten Texten

  2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

    1. Die Evolvierbarkeit vom WWW
    2. Ökosystem des Webs ist seine größte Stärke
    3. Benötigte Zentrale Kontrolle, falls man referentielle Integrität verlangt - verlangsamt Wachstum

  3. Was wäre der Preis für die garantierte Verhinderung von “broken links”?

    1. Eine zentrale Kontrolle wäre notwendig. Dies würde eine starke Verringerung der Evolvierbarkeit bedeuten

1.2. HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

    1. Alles funktioniert

  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

    1. 301 Moved Permanently bedeutet, dass die Ressource permanent zu einer anderen URL getan wurde

  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

  4. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
